# MarkWay

> 讓 AI Agent 以清晰、智能的方式瀏覽網頁。

## 為什麼選擇 MarkWay？

網際網路是為人而建的，而非機器。人類可以輕鬆解析 HTML、瀏覽複雜的佈局、理解視覺層次結構，但 AI Agent 在這些任務上卻舉步維艱。MarkWay 透過提供一種協定來彌合這一差距，這種協定以 AI 最理解的語言說話：**結構化、語義化的內容**。

## 問題所在

傳統的網頁瀏覽要求 AI Agent：
- 解析複雜的 HTML 結構
- 執行 JavaScript 來渲染內容
- 處理不一致的類別名稱和 DOM 結構
- 應對反爬蟲措施和速率限制
- 從視覺佈局中擷取有意義的資料

這種方法脆弱、低效且計算成本高昂。

## 解決方案

MarkWay 引入了一種 **Markdown 優先的協定**，為 AI Agent 改變了網路體驗：

```
網頁 → Markdown → Agent
```

透過直接以 Markdown 格式提供內容，MarkWay 實現了：

- **語義清晰**：內容結構明確無歧義
- **高效解析**：無需 HTML 解析開銷或 DOM 操作
- **自然理解**：AI 像人類閱讀文件一樣理解內容
- **協定發現**：內建機制讓 Agent 理解站點結構

## 工作原理

MarkWay 以兩種模式運作：

### 靜態模式
內容以 Markdown 檔案組織，具有可預測的目錄結構：
- 每個目錄包含一個 `index.md`，帶有結構化的目錄表
- 所有連結以 `.md` 結尾以保持一致性
- 支援相對路徑、絕對路徑和外部路徑

### 動態模式
互動式端點語義化使用 HTTP 方法：
- `GET` 返回 Markdown 文件（無 URL 參數）
- `POST` 透過請求體執行資料交換
- 透過 `Accept` 頭協商回應格式

## 快速開始

對於站點所有者：
1. 以 Markdown 格式提供您的內容
2. 在每個目錄中包含 `index.md`，列出檔案及其用途的表格
3. 根請求必須連結到 `/protocol` 的協定文件

對於 AI Agent 開發者：
1. 透過 `/protocol` 或 `/protocol.md` 的協定發現檢測 MarkWay 站點
2. 解析 `index.md` 表格以導航站點結構
3. 使用 `GET` 取得文件，使用 `POST` 進行資料交換

## 使用場景

- **API 文件**：人類可讀且 Agent 可完美解析的文件
- **知識庫**：人類和 AI 都可存取的結構化資訊
- **資料服務**：具有自文件化介面的查詢端點
- **AI 原生應用**：專為 Agent 消費而建構的網路服務
