# MarkWay

> 让 AI Agent 以清晰、智能的方式浏览网页。

## 为什么选择 MarkWay？

互联网是为人而建的，而非机器。人类可以轻松解析 HTML、浏览复杂的布局、理解视觉层次结构，但 AI Agent 在这些任务上却举步维艰。MarkWay 通过提供一种协议来弥合这一差距，这种协议以 AI 最理解的语言说话：**结构化、语义化的内容**。

## 问题所在

传统的网页浏览要求 AI Agent：
- 解析复杂的 HTML 结构
- 执行 JavaScript 来渲染内容
- 处理不一致的类名和 DOM 结构
- 应对反爬虫措施和速率限制
- 从视觉布局中提取有意义的数据

这种方法脆弱、低效且计算成本高昂。

## 解决方案

MarkWay 引入了一种 **Markdown 优先的协议**，为 AI Agent 改变了网络体验：

```
网页 → Markdown → Agent
```

通过直接以 Markdown 格式提供内容，MarkWay 实现了：

- **语义清晰**：内容结构明确无歧义
- **高效解析**：无需 HTML 解析开销或 DOM 操作
- **自然理解**：AI 像人类阅读文档一样理解内容
- **协议发现**：内置机制让 Agent 理解站点结构

## 工作原理

MarkWay 以两种模式运行：

### 静态模式
内容以 Markdown 文件组织，具有可预测的目录结构：
- 每个目录包含一个 `index.md`，带有结构化的目录表
- 所有链接以 `.md` 结尾以保持一致性
- 支持相对路径、绝对路径和外部路径

### 动态模式
交互式端点语义化使用 HTTP 方法：
- `GET` 返回 Markdown 文档（无 URL 参数）
- `POST` 通过请求体执行数据交换
- 通过 `Accept` 头协商响应格式

## 快速开始

对于站点所有者：
1. 以 Markdown 格式提供您的内容
2. 在每个目录中包含 `index.md`，列出文件及其用途的表格
3. 根请求必须链接到 `/protocol` 的协议文档

对于 AI Agent 开发者：
1. 通过 `X-MarkWay-Protocol` 头或协议发现检测 MarkWay 站点
2. 解析 `index.md` 表格以导航站点结构
3. 使用 `GET` 获取文档，使用 `POST` 进行数据交换

## 使用场景

- **API 文档**：人类可读且 Agent 可完美解析的文档
- **知识库**：人类和 AI 都可访问的结构化信息
- **数据服务**：具有自文档化接口的查询端点
- **AI 原生应用**：专为 Agent 消费而构建的网络服务
